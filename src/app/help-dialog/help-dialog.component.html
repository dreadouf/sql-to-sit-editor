Overview

This tool allows the generation of SIT queries from an ODS SQL query with or without DW query. The ODS query MUST follow certain requirements!

ODS SQL Query Requirements

  • Every column in the main (first) SELECT must have an alias.
  • Preface the first indicator column by --#! (Check the following example)

How this tool works?

  1. The input is: ODS Query, DW table name, and the validation indicator.
  2. Extract from the main SELECT of the ODS query the columns (with their aliases).
  3. Generate the DW query from the ODS columns' aliases and DW table name (if the DW query is not indicated by the user).
